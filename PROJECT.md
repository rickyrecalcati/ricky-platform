# Ricky Recalcati Platform

# Definition of Done

A feature is complete only if:

- Builds successfully
- Passes lint
- Uses existing design language
- Works on mobile
- Is accessible
- Does not break existing routes
- Does not duplicate code
- Uses reusable components where appropriate
- Includes a summary of changed files

# Quality Standard

Every page should be good enough to appear in a gallery of the best designed author websites.

If a feature feels average, redesign it.

Always prefer quality over speed.

---

# Coding Rules

Keep files organised.

Prefer reusable components.

Never duplicate layouts.

Never modify unrelated files.

Always explain changed files after completing a task.

Always run build before finishing.
